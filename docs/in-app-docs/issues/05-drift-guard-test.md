## What to build

The lightweight integrity test that keeps the docs honest with the code: assert that
every inline `<Hint>` slug resolves to a real glossary entry and that intra-doc links
resolve. This is the cheap guard from design principle #5 (trust through precision) — it
catches broken popovers and dead deep-links, the most visible failure mode. Best landed
before slice 04's big content pass so it guards that work.

End-to-end demo: the test passes on the current tree; renaming or removing a glossary slug
that a `<Hint>` still references turns the test red.

Scope is deliberately narrow — defer the heavier "does this documented field still exist
in the form / DB schema" cross-check.

## Acceptance criteria

- [ ] A unit test (web-admin test runner) that collects every `<Hint slug="…">` usage and asserts each `slug` exists in the glossary data file
- [ ] The test asserts intra-doc links (hub page links + `#/docs/concepts/:slug` deep-link targets) resolve to real pages/anchors
- [ ] The test fails clearly (names the offending slug/link) when a referenced term or page is missing
- [ ] Wired into the CI test step so drift breaks the build
- [ ] Does NOT attempt the field-existence / schema cross-check (explicitly out of scope for v1)
- [ ] Build/lint/typecheck/test pass

## Triage note — test harness

> *This was generated by AI during triage.*

Resolved the open question on "web-admin test runner" / "wired into the CI test step":

- web-admin has **no test runner today** (`apps/web-admin/package.json` scripts: `dev`, `build`, `preview`, `check` only — no `test`, no vitest).
- The repo standard is **`bun test`**: root `package.json` has `"test": "bun test"` and CI runs `bun run test` (`.github/workflows/ci.yml:29`), which discovers `*.test.ts` repo-wide.
- **No new harness is needed.** Add the drift test as a plain `bun:test` `*.test.ts` under `apps/web-admin/` (e.g. `src/docs/drift.test.ts`); the existing root `bun run test` auto-discovers it → already in CI. Do **not** add vitest or a per-app `test` script.
- **Gotcha:** `bun test` cannot import `.svelte` files (no Svelte transform). The test must **read `.svelte` source as text** and regex-scan for `<Hint slug="…">` usages, and **import the glossary `.ts`** directly (Bun loads `.ts` natively). Keep it source-scanning, not component-mounting.
