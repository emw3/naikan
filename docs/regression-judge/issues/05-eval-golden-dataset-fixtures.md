# 05 — Eval golden-dataset fixtures (+ demo seed)

Status: ready-for-agent
Category: enhancement
Type: AFK

## Parent

`docs/regression-judge/PRD.md`

## What to build

The labeled corpus the judge is measured against — and, by reuse, the one-command
demo seed. A **hybrid** dataset: mostly **synthetic** cases generated by capturing
deterministic seed pages through the **real** UI pipeline (so the artifacts are
genuine **Diff**s, not mocks), plus a few **curated real** cases for credibility.
Each fixture carries a baseline screenshot, a current screenshot, the diff overlay +
diff%, the run's **Signals**, and a ground-truth label.

The synthetic generator doubles as the demo seed: it leaves the local stack holding a
project with a real regression and a noise diff, ready for the slice-03 agent to
judge live.

**End-to-end demo:** run the seed/generator against the local stack → a Project with
UI checks whose runs include a genuine layout break (`real_regression`), a
dynamic-content diff (`noise`), and an intentional restyle (`intentional`); the
fixture set is written to a known location with labels.

## Acceptance criteria

- [ ] Deterministic seed pages: at least one that **genuinely breaks layout** (→ `real_regression`), one with **dynamic content** that diffs but isn't a break — rotating carousel / live timestamp (→ `noise`), and one **intentional restyle** (→ `intentional`)
- [ ] A generator/seed script captures these through the real capture + diff pipeline (baseline promoted, page changed, re-captured) — produces genuine baseline / current / diff artifacts + Signals, not hand-mocked images
- [ ] Synthetic fixtures are labeled by construction (the script knows what it changed); labels recorded in a manifest alongside the artifacts at a known path
- [ ] A few (≥3) **curated real** cases included, each with a human-confirmed ground-truth label — this label pass is the one human-in-the-loop step; mark those fixtures as `source: curated`
- [ ] The dataset format is documented (what each fixture contains + the label vocabulary) so the slice-06 harness can consume it
- [ ] The synthetic seed is reusable as the local **demo seed** (idempotent / re-runnable against a fresh stack)
- [ ] existing `bun test` passes (the generator itself need not be unit-tested; it exercises the real pipeline)

## Blocked by

None — uses the existing capture / diff / baseline pipeline (worker, `@naikan/ui-runner`, `@naikan/baseline-store`). Independent of the MCP slice.
