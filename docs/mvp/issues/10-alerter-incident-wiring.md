## What to build

The `alerter` module dispatching alerts to email and Slack. Email via **Resend** (resend.com — decided in triage, recorded in ADR-0003). Slack via incoming webhook URL on the Project config.

Wire incident transitions to alerts:
- `opened` → "Incident opened for <project> / <check>" with current run error
- `closed-recovered` → "Incident recovered after <duration>" with downtime

Per-project routing honoured: each Project has independent `alert_emails[]` and `slack_channel` (i.e. webhook URL).

End-to-end demo: a deliberately-broken HeartbeatCheck breaches its `alert_after_n_fails`, an email + Slack message arrives at the project's configured channels; once it recovers, a recovery alert with duration arrives.
