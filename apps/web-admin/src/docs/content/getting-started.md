# Getting started

Welcome to the **Naikan Monitor** documentation. This is the in-app home for
understanding every field, form, check, and status you'll meet across the admin —
written for the team, not for projects.

## What this is

Naikan Monitor watches your projects' sites and tells you, at a glance, whether each
one is healthy. It runs two kinds of automated checks:

- **Heartbeat checks** — is the site reachable and responding the way it should?
- **UI checks** — does the page still *look* right? We capture a screenshot, compare it
  against an approved baseline, and surface the visual diff.

When something drifts, the monitor raises a signal so you can act before a project does.

## Finding your way around

The sidebar is the map:

- **Overview** — the at-a-glance health of everything you can see.
- **Projects** — drill into a single project, its sites, and every check on them.
- **Incidents** — what's failing right now, across all projects.
- **Docs** — you're here.

## How docs work

This hub is the single source of truth. As we roll docs out, you'll start seeing small
`?` affordances next to fields and columns across the app — each one opens a short
explanation and links back *into* this hub. The copy lives in one place, so what you
read inline always matches what you read here.

The hub is organised into three sections:

- **Concepts** — the vocabulary: what a *baseline*, a *diff*, or a *signal* actually is.
- **Guides** — how to do a thing, step by step.
- **Reference** — the exhaustive detail when you need it.

More content lands here as we go. Start with **Concepts** when it's ready to build a
shared vocabulary for the rest of the app.
